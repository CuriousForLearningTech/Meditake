# MediTake Documentation

This folder contains the project documentation for **MediTake**, a medication adherence platform designed to help people manage long-term treatment plans.

## What To Read First

1. [Backend Architecture](./backend-architecture.md)
2. [Architecture Diagram](./Microservice_Architecture_Meditak.png)

## Project Snapshot

- **Product:** MediTake
- **Purpose:** medication reminders, prescription management, and adherence support
- **Current phase:** active development and prototyping
- **Backend direction:** microservice-based design with a shared prototype database

## Project At A Glance

- **Repository layout:** backend, mobile, and web apps in one monorepo
- **Mobile app:** Flutter
- **Web app:** Next.js and TypeScript
- **Backend:** FastAPI and Python
- **Data layer:** PostgreSQL for the current prototype

## Documentation Goals

This documentation explains:

- why the backend is split into services
- how requests flow through the platform
- which parts are implemented now and which are planned for later
- the future direction for scaling, security, and operations

## Notes

- The architecture shown in this folder is not treated as a final production design.
- Some parts are intentionally marked as future work because the product is still being built.
